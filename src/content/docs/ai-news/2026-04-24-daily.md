## 开源热门
> 数据来源：[Trendshift](https://trendshift.io/) · [GitHub Trending](https://github.com/trending)

### [Gitlawb/openclaude](https://github.com/Gitlawb/openclaude)
**⭐ 23,996 · TypeScript**

一个支持 OpenAI、DeepSeek、Ollama 等 200+ 模型的开源终端 AI 编程助手。该项目为开发者提供了平替 Claude Code 的本地化方案，非常适合需要接入私有模型或进行多模型代码生成对比测试的工程团队。

### [addyosmani/agent-skills](https://github.com/addyosmani/agent-skills)
**⭐ 20,835 · Shell**

面向 AI 编程 Agent 的生产级工程技能库。它提供了一套标准化的底层系统交互脚本集，能显著降低开发者为自定义 Agent 编写文件操作、环境检测等基础工具（Tools）的开发成本。

### [zilliztech/claude-context](https://github.com/zilliztech/claude-context)
**⭐ 8,299 · TypeScript**

专为 Claude Code 设计的代码搜索 MCP（模型上下文协议）服务器。它解决了大型代码库中 Agent 缺乏全局上下文的痛点，是提升本地代码库 RAG 质量和跨文件重构准确率的实用组件。

### [forrestchang/andrej-karpathy-skills](https://github.com/forrestchang/andrej-karpathy-skills)
**⭐ 72,645 · Markdown**

提炼自 Andrej Karpathy 观察的 LLM 编程避坑指南，封装为单一的 CLAUDE.md 文件。通过系统提示词工程规范了 Agent 的行为边界，为团队制定 AI 辅助编程规范和 System Prompt 提供了极佳的参考模板。

## 论文精选

### DR-Venus：仅用 1 万条开源数据构建边缘端深度研究 Agent
**👍 38** · [arXiv](https://arxiv.org/abs/2604.19859)

本文探讨了如何通过提升数据质量和利用率，仅用 1 万条开源数据训练出一个强大的 4B 参数边缘端深度研究 Agent。这为在成本、延迟和隐私受限的真实场景下，低成本合成数据并部署本地化小模型 Agent 提供了极具价值的工程范式。

### OpenMobile：通过任务与轨迹合成构建开源移动端 Agent
**👍 23** · [arXiv](https://arxiv.org/abs/2604.15093)

该论文提出了一个开源框架，用于合成移动端 Agent 的任务和操作轨迹数据，打破了现有顶尖视觉语言模型在训练数据上的封闭性。对于需要开发跨应用自动化工具、RPA Agent 或多模态工具调用的工程师来说，其数据合成与环境交互策略极具启发性。

### 近未来策略优化（Near-Future Policy Optimization）
**👍 45** · [arXiv](https://arxiv.org/abs/2604.20733)

文章提出了一种通过引入合适的离策略（off-policy）轨迹来加速强化学习（RLVR）收敛的方法，解决了后训练阶段高质量探索轨迹难以获取的痛点。这对当前致力于通过强化学习提升大模型复杂推理能力（如复现 O1 类似架构）的算法工程师具有直接的指导意义。
