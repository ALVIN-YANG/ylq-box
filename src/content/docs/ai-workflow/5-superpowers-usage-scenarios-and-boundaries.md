# superpowers：使用方式、适用场景和边界

先说结论：**superpowers 值得装，但更适合当重型模式，不适合覆盖所有日常任务。**

官方链接：

- 仓库：<https://github.com/obra/superpowers>
- OpenCode 安装入口：<https://raw.githubusercontent.com/obra/superpowers/refs/heads/main/.opencode/INSTALL.md>

## 1. 最短上手

```text
先帮我把这个需求边界和实现方案梳理清楚，不要急着写代码。
```

如果方案清楚了，再接：

```text
方案确认后，拆成执行步骤，然后按步骤完成，做完跑测试和构建。
```

## 2. 它解决什么问题

它解决的核心问题是：

- 需求没盘清楚就写代码
- 改到一半发现方向不对
- 写完了没 review、没验证、没收尾

## 3. 直接抄走的提示词

### 需求不清，先别写代码

```text
先帮我把这个需求边界、风险和实现方案梳理清楚，不要急着写代码。
```

### 方案确认后拆计划

```text
方案确认了，按执行顺序拆成几个小任务，每个任务写清楚目标和验证方式。
```

### 修一个复杂 bug

```text
先确认复现条件，再定位根因，再给修复方案，最后完成修复和验证。
```

## 4. 最后一句话

```text
AI Coding 工具负责干活，superpowers 负责让这件事更像一个靠谱团队在干活。
```
