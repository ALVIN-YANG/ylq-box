# [跨界玩法] 不只是写代码：打造图文并茂的公众号/博客自动化流水线

这是本专栏的最后一篇。我们不谈枯燥的 Git，不谈 CI/CD，也不谈如何手撸红黑树。

作为一名技术人员，你的副业可能是经营一个个人博客、一个知乎账号、或者是一个微信公众号。写代码可能只要几小时，但构思大纲、查阅最新的外网资料、配图、排版、转成 Markdown、推送到 GitHub Pages……这些运营琐事往往会耗费你一整天。

而今天，我将向你展示：**如何利用 Opencode 的工具链（Toolchain），将这个枯燥的内容生产流水线压缩到只需你敲一句话（一键执行）。**

## 1. 原理拆解：为什么 Opencode 能做自媒体？

Opencode 的核心是一个具备文件操作与系统调度能力的智能代理。要写一篇带有热点资讯和配图的文章，它需要以下工具库的配合：

1. **信息获取**：使用 `webfetch` 或者 MCP 的浏览器控制插件，访问 Hacker News、Reddit 或 Twitter 获取最新的 AI 新闻或技术热点。
2. **深度思考与整合**：大模型对获取到的全英文噪音网页进行清洗、翻译、归纳，提取出 3 个核心看点。
3. **多模态配图生成（或搜图）**：通过调用系统脚本或特定的绘图 API（如 Midjourney 或 DALL-E 3 MCP 插件），根据提取的摘要生成相关文章配图。
4. **Markdown 渲染与排版**：大模型将文本与图片链接按特定的规范（如 Astro/Hexo/Hugo 的 frontmatter 规范）组合成完整的 Markdown 文档。
5. **自动发布**：调用本地的 `bash` 运行 Git 提交并推送到远端服务器触发自动构建。

## 2. 实战演练：让 Opencode 自动帮我们写“周报”

假设你想在你的 Astro 博客上更新一篇关于《本周 AI 前沿技术简报》的文章。你只需要给 Opencode 这样一个 Prompt：

> "请访问 Hacker News，找到今天热度最高的 3 篇关于人工智能的技术文章。
> 阅读它们的正文，然后用中文写一篇名为《AI 技术早报》的 Markdown 博文。
> 对于每一个新闻，请自动在 Unsplash（免费图库）上生成一个相关的搜索链接作为配图插入。
> 最后将这篇 Markdown 写入到 `src/content/docs/ai-news/today.md` 中。并帮我 Git Commit。"

### 幕后的魔法是怎样发生的？

1. **爬虫启动**：Opencode 立刻通过 `bash` 调用了如 `curl`，或者是通过内置的 `webfetch` 拿下了 Hacker News 的网页源码，并从中提取到了目标 URL。
2. **递归阅读**：接着，它深入每一个新闻 URL 进行二次爬取。模型强大的上下文能够瞬间阅读并理解三篇各长达数千字的英文原贴。
3. **组装 Markdown**：它开始思考排版，写出优雅的中文导读。在配图时，它利用逻辑推理，比如第一篇新闻是关于机器人的，它会插入 `![Robot](https://source.unsplash.com/800x400/?robot,ai)` 这个动态获取图片的链接。
4. **落地文件**：调用内置的 `write` 工具，确保文件结构完美符合你的博客系统要求（包含 YAML 头信息）。
5. **版本控制**：最后调用 `bash` 敲下 `git add . && git commit -m "docs: add AI daily news" && git push`。

两分钟后，你的手机上收到了网站更新成功的推送通知。

## 3. 进阶：如何处理微信公众号那变态的排版？

如果你做过公众号就知道，公众号的编辑器简直是反人类的，它不支持原生的 Markdown。

Opencode 怎么解决？
在你的工作流中，你可以让 Opencode 不仅仅生成 `.md`。你可以写一个指令，让它调用本地的 `md2wx` 这种命令行转换工具，将 Markdown 转化为内联了 CSS 样式的 HTML。

更绝的是，如果你在本地配置了微信官方的自动化上传脚本（比如基于 Puppeteer 封装的脚本），Opencode 甚至能在生成完 HTML 后，直接执行 `npm run upload-to-wechat`，连草稿都帮你存在了后台，你只用掏出手机点击“群发”。

## 4. 人工智能不是替代品，是杠杆

这是我写在 Opencode 专栏结尾的话。

很多人在尝试了几次 ChatGPT 后，觉得“AI 写出来的东西没有灵魂，AI 写的代码一堆 Bug”，然后转身回到了手工作坊时代。

但这是因为他们**使用 AI 的姿势错了**。

如果你把 AI 当成一个神灵，指望一句话就能让它凭空造出完美的艺术品或工程，你一定会失望。
真正的极客，是把 AI 视为一个**“可以理解复杂意图的调度器”**。

在 Opencode 的世界里：
- 我们给它装上 `grep` 和 `glob` 的眼睛，让它不再瞎猜代码。
- 我们给它插上 LSP 的大脑，让它懂得精准的类型定义。
- 我们给它配上 MCP 的雷达，让它能感知远端的数据库和 Jira 流水线。
- 最后，我们用 CI/CD 的规章制度把它圈在沙盒里，让它安全、稳定地在流水线的最前端为你开疆拓土。

**不要去对抗自动化，去拥抱那个能帮你敲命令的 CLI 数字实习生。**

属于“意图驱动编程（Intent-Driven Programming）”的时代，已经伴随着 Opencode 的敲击声，真正到来了。
