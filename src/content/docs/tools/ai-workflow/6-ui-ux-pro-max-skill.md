# ui-ux-pro-max-skill：把 AI 从会写页面，推到会做 UI/UX 方案

先说结论：**这个 skill 的价值，不是让 AI 生成一套更花的界面，而是逼 AI 先把 UI 方向、UX 路径、组件规则和实现计划讲清楚。**

项目地址：

- GitHub：<https://github.com/nextlevelbuilder/ui-ux-pro-max-skill>

## 1. 它解决的是什么问题

很多 AI 做前端时，问题不是不会写代码，而是：

1. 一上来就写组件
2. 不先问平台、目标、约束
3. 把 UI、UX、设计系统、实现混成一锅
4. 只给一个页面，不给状态、不讲交互、不讲边界

## 2. 直接抄走的提示词

### 做页面方案

```text
用 ui-ux-pro-max 帮我做一个产品页的 UI/UX 方案。
先给视觉方向、布局结构、关键模块、空状态和实现计划，不要直接写代码。
```

### 做管理后台

```text
用 ui-ux-pro-max 设计一个 React + Tailwind 的管理后台。
输出 UI 方向、信息架构、关键组件、loading/empty/error 状态，以及文件级实现计划。
```

### 做设计系统

```text
用 ui-ux-pro-max 帮我先定义这套产品的设计系统。
给颜色、字体、间距、圆角、阴影、按钮和表单规则，再给页面层应用建议。
```

## 3. 最后一句话

```text
ui-ux-pro-max-skill 最有价值的地方，不是让 AI 更会画页面，而是让 AI 先把设计工作拆清楚。
```
